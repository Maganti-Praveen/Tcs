Q2. JSX — JavaScript XML
JSX lets you write HTML-like syntax in JavaScript. It compiles to React.createElement() calls.
💻 Code
// JSX
const element =
  <h1 className="title">
    Hello World
  </h1>;

// Compiles to:
React.createElement(
  'h1',
  { className: 'title' },
  'Hello World'
);

// JSX Rules:
// 1. Return single parent element
// 2. className (not class)
// 3. htmlFor (not for)
// 4. camelCase attributes
// 5. Self-close tags: <img />
// 6. Wrap in () for multiline
📋 JSX Expressions
const name = "Amit";
const age = 25;

// Variables in JSX
const el = <h1>Hi {name}</h1>;

// Expressions
const el2 = <p>{age * 2}</p>;

// Ternary
const el3 =
  <p>{age > 18 ? "Adult" : "Minor"}
  </p>;

// Function calls
const el4 =
  <p>{name.toUpperCase()}</p>;

// ❌ Cannot use:
// if/else, for loops, switch
// inside JSX curly braces.
// Use ternary or map() instead.

Q4. Class Component (Legacy — Know for Interviews)
Class components are legacy but still asked in interviews. Know the syntax but use functional components in practice.
💻 Code
import React, { Component }
  from 'react';

class Greeting extends Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  increment = () => {
    this.setState({
      count: this.state.count + 1
    });
  };

  render() {
    return (
      <div>
        <h1>{this.props.name}</h1>
        <p>Count: {this.state.count}
        </p>
        <button
          onClick={this.increment}>
          +1
        </button>
      </div>
    );
  }
}

export default Greeting;
📋 Class vs Functional
// CLASS:
// this.state, this.setState()
// this.props
// render() method
// Lifecycle methods
// constructor, super(props)

// FUNCTIONAL (Modern ✅):
// useState() hook
// props as parameter
// return JSX directly
// useEffect() for lifecycle
// No constructor needed

// INTERVIEW ANSWER:
// "I use functional components
//  with hooks because they're
//  simpler, cleaner, and the
//  recommended approach by
//  the React team."

Q7. Passing Functions as Props (Child → Parent)
To send data from child to parent, pass a function as a prop. This is called "lifting state up".
💻 Code
// ---- Parent ----
const App = () => {
  const handleDelete = (id) => {
    console.log("Delete user:", id);
  };

  return (
    <UserCard
      name="Amit"
      id={1}
      onDelete={handleDelete}
    />
  );
};

// ---- Child ----
const UserCard = ({ name, id,
  onDelete }) => {
  return (
    <div>
      <h2>{name}</h2>
      <button
        onClick={() => onDelete(id)}>
        Delete
      </button>
    </div>
  );
};
📋 Notes
// When "Delete" clicked:
// → Child calls onDelete(1)
// → Parent's handleDelete runs
// → Console: "Delete user: 1"

// Data flow in React:
// Props: Parent → Child (down)
// Events: Child → Parent (up)

// Common pattern:
// Parent holds state
// Parent passes data + functions
// Child calls parent function
// Parent updates state
// React re-renders

// ❌ onClick={onDelete(id)}
//    → calls immediately!
// ✅ onClick={() => onDelete(id)}
//    → calls only on click

Section 4 — State & useState 🔥

Q8. useState — Basic State
State is mutable data that belongs to a component. When state changes, the component re-renders.
💻 Code
import { useState } from 'react';

const Counter = () => {
  const [count, setCount] =
    useState(0);

  return (
    <div>
      <h1>Count: {count}</h1>
      <button
        onClick={() =>
          setCount(count + 1)}>
        Increment
      </button>
      <button
        onClick={() =>
          setCount(count - 1)}>
        Decrement
      </button>
      <button
        onClick={() =>
          setCount(0)}>
        Reset
      </button>
    </div>
  );
};
📋 Notes
Count: 0
[Increment] [Decrement] [Reset]

// useState(initialValue)
// Returns: [value, setter]

// const [count, setCount] =
//   useState(0);
// count     → current value
// setCount  → function to update

// Rules:
// ✅ Call at top of component
// ✅ Never inside if/for/nested
// ✅ setCount triggers re-render
// ❌ count = 5 (never mutate!)
// ✅ setCount(5) (use setter)

// State vs Props:
// State → owned by component
//   → mutable (via setter)
// Props → passed from parent
//   → read-only (immutable)
Q9. useState — Object & Array State
💻 Code
// Object State
const [user, setUser] = useState({
  name: "Amit",
  age: 25,
  city: "Delhi"
});

// Update one field (spread!)
setUser({
  ...user,
  age: 26
});
// ❌ setUser({ age: 26 })
//    → loses name and city!

// Array State
const [todos, setTodos] = useState([
  { id: 1, text: "Learn React" },
  { id: 2, text: "Build project" }
]);

// Add item
setTodos([...todos,
  { id: 3, text: "Deploy" }]);

// Remove item
setTodos(
  todos.filter(t => t.id !== 2));

// Update item
setTodos(todos.map(t =>
  t.id === 1
    ? { ...t, text: "Done!" }
    : t
));
📋 Key Rules
// ALWAYS create new reference!
// React compares references to
// detect changes.

// ❌ WRONG (mutation):
// user.age = 26;
// setUser(user);
// → Same reference, no re-render!

// ✅ CORRECT (new object):
// setUser({ ...user, age: 26 });
// → New reference, re-renders!

// Array operations:
// Add:    [...arr, newItem]
// Remove: arr.filter(x => x.id !== id)
// Update: arr.map(x =>
//           x.id === id ? {...} : x)

// ❌ NEVER use:
// push(), pop(), splice()
// These mutate the original array!

// ✅ ALWAYS use:
// spread, filter, map, concat
// These return NEW arrays!
Q10. Functional Updates (Previous State)
💻 Code
// ❌ Problem with direct value:
const handleClick = () => {
  setCount(count + 1);
  setCount(count + 1);
  setCount(count + 1);
  // Only increments by 1!
  // All 3 use same stale count
};

// ✅ Solution: functional update
const handleClick = () => {
  setCount(prev => prev + 1);
  setCount(prev => prev + 1);
  setCount(prev => prev + 1);
  // Increments by 3! ✅
  // Each uses latest prev value
};
📋 Notes
// Use functional form when:
// → New state depends on old
// → Multiple updates in handler
// → Inside setTimeout/intervals

// setCount(count + 1)
//   → uses stale closure value
// setCount(prev => prev + 1)
//   → uses latest value ✅

// INTERVIEW QUESTION:
// "Why does clicking +3
//  only add 1?"
// → Because React batches
//   state updates, and all 3
//   setCount calls see the
//   same count value.
// → Use prev => prev + 1

Q12. Conditional Rendering
💻 Code
const Dashboard = ({ isLoggedIn,
  isAdmin, notifications }) => {
  return (
    <div>
      {/* Ternary */}
      {isLoggedIn
        ? <h1>Welcome!</h1>
        : <h1>Please Login</h1>
      }

      {/* AND operator */}
      {isAdmin &&
        <button>Admin Panel
        </button>
      }

      {/* Notifications badge */}
      {notifications > 0 &&
        <span>({notifications})
        </span>
      }

      {/* Multiple conditions */}
      {isLoggedIn && isAdmin

        : isLoggedIn
          ? <p>Limited Access</p>
          : <p>No Access</p>
      }
    </div>
  );
};
📋 Notes
// 3 Ways to Conditionally Render:

// 1. Ternary (? :)
// → When you have if/else
// {cond ? <A/> : <B/>}

// 2. AND (&&)
// → When you only have if
// {cond && <A/>}

// 3. Early return
const Comp = ({ data }) => {
  if (!data) return <p>Loading</p>;
  return <p>{data.name}</p>;

Section 8 — useEffect (Side Effects) 🔥

🔥 useEffect is the #1 most asked React hook in interviews. It handles side effects: API calls, subscriptions, timers, DOM manipulation.
Q16. useEffect — Basics & Dependency Array
💻 Code
import { useState, useEffect }
  from 'react';

const Demo = () => {
  const [count, setCount] =
    useState(0);

  // 1. Runs on EVERY render
  useEffect(() => {
    console.log("Every render");
  });

  // 2. Runs ONCE (on mount)
  useEffect(() => {
    console.log("Mounted!");
  }, []);  // ← empty array

  // 3. Runs when count changes
  useEffect(() => {
    console.log("Count:", count);
  }, [count]);  // ← dependency

  // 4. Cleanup (unmount)
  useEffect(() => {
    const timer = setInterval(
      () => console.log("tick"),
      1000
    );
    return () => {
      clearInterval(timer);
      console.log("Cleaned up!");
    };
  }, []);

  return (
    <div>
      <p>{count}</p>
      <button onClick={() =>
        setCount(c => c + 1)}>
        +1</button>
    </div>
  );
};
📋 Rules
// Dependency Array Controls WHEN:

// useEffect(() => {}, )
//   → No array → every render

// useEffect(() => {}, [])
//   → Empty [] → only on mount

// useEffect(() => {}, [count])
//   → [dep] → when dep changes

// Cleanup function:
// return () => { ... }
// → Runs on unmount
// → Also runs before re-run
// → Used for: timers, sockets,
//   subscriptions, event listeners

// Lifecycle mapping:
// componentDidMount → useEffect(,[])
// componentDidUpdate → useEffect(,[dep])
// componentWillUnmount → cleanup

// ⚠️ Common mistake:
// Missing dependency → stale data
// Extra dependency → infinite loop
Q17. useEffect — API Call (Fetch Data)
This is the most common real-world usage of useEffect — fetching data on component mount.
💻 Code
const UserList = () => {
  const [users, setUsers] =
    useState([]);
  const [loading, setLoading] =
    useState(true);
  const [error, setError] =
    useState(null);

  useEffect(() => {
    const fetchUsers = async () => {
      try {
        const res = await fetch(
          '/api/users');
        if (!res.ok)
          throw new Error('Failed');
        const data =
          await res.json();
        setUsers(data);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchUsers();
  }, []); // Run once on mount

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error}</p>;

  return (
    <ul>
      {users.map(u => (
        <li key={u._id}>{u.name}</li>
      ))}
    </ul>
  );
};
📋 Pattern
// Standard fetch pattern:
// 1. State: data, loading, error
// 2. useEffect with async function
// 3. try/catch/finally
// 4. Conditional rendering

// Loading → true → show spinner
// Success → set data → render list
// Error → set error → show message

// ⚠️ Can't make useEffect async:
// ❌ useEffect(async () => {})
// ✅ Define async inside, call it

// With Axios (cleaner):
// const res = await axios.get(url);
// setUsers(res.data);

// Cleanup for abortable fetch:
useEffect(() => {
  const controller =
    new AbortController();
  fetch(url, {
    signal: controller.signal
  });
  return () => controller.abort();
}, []);

Q18. useRef — DOM Access & Persistent Values
💻 Code
import { useRef, useEffect }
  from 'react';

const InputFocus = () => {
  const inputRef = useRef(null);
  const renderCount = useRef(0);

  // Focus input on mount
  useEffect(() => {
    inputRef.current.focus();
  }, []);

  // Track renders (no re-render)
  useEffect(() => {
    renderCount.current += 1;
  });

  return (
    <div>
      <input ref={inputRef}
        placeholder="Auto-focused"
      />
      <p>Renders:
        {renderCount.current}</p>
    </div>
  );
};
📋 Notes
// useRef returns:
// { current: value }

// Two main uses:

// 1. Access DOM elements
// ref={inputRef}
// inputRef.current.focus()
// inputRef.current.value
// inputRef.current.scrollTo()

// 2. Store mutable values
// That persist across renders
// WITHOUT causing re-render
// (unlike useState)

// useRef vs useState:
// useState → re-renders on change
// useRef   → does NOT re-render

// Common uses:
// Auto-focus input
// Scroll to element
// Store previous value
// Store timer IDs

Q19. useMemo — Expensive Computations
💻 Code
import { useState, useMemo }
  from 'react';

const ExpensiveList = ({ items }) => {
  const [query, setQuery] =
    useState('');

  // Without useMemo:
  // Filters on EVERY re-render
  // Even when query didn't change!

  // With useMemo:
  // Only re-computes when
  // items or query changes
  const filtered = useMemo(() => {
    console.log("Filtering...");
    return items.filter(item =>
      item.name.toLowerCase()
        .includes(query.toLowerCase())
    );
  }, [items, query]);

Q20. useCallback — Memoize Functions
💻 Code
import { useState, useCallback }
  from 'react';

const Parent = () => {
  const [count, setCount] =
    useState(0);

  // Without useCallback:
  // New function every render
  // → Child re-renders always!

  // With useCallback:
  // Same function reference
  // → Child skips re-render
  const handleDelete =
    useCallback((id) => {
      console.log("Delete:", id);
    }, []);

  return (
    <div>
      <p>{count}</p>
      <button onClick={() =>
        setCount(c => c + 1)}>
        +1</button>
      <ChildList
        onDelete={handleDelete} />
    </div>
  );
};

// Wrap child with React.memo
const ChildList = React.memo(
  ({ onDelete }) => {
    console.log("Child rendered");
    return <button
      onClick={() =>
        onDelete(1)}>
      Delete</button>;
});
📋 Notes
// useCallback(fn, [deps])
// → Caches the FUNCTION itself
// → Same reference unless deps change

// Pair with React.memo:
// React.memo → skips re-render
//   if props haven't changed
// useCallback → ensures function
//   prop has same reference

// Without useCallback:
// Parent re-renders → new function
// → New reference → Child re-renders

// With useCallback:
// Parent re-renders → same function
// → Same reference → Child skips!

// Summary:
// useMemo   → cache computed VALUE
// useCallback → cache FUNCTION
// React.memo  → cache COMPONENT
Q21. useReducer — Complex State Logic
useReducer is like useState but for complex state with multiple actions. Similar to Redux pattern.
💻 Code
import { useReducer } from 'react';

const initialState = {
  count: 0,
  step: 1
};

const reducer = (state, action) => {
  switch (action.type) {
    case 'INCREMENT':
      return {
        ...state,
        count: state.count
          + state.step
      };
    case 'DECREMENT':
      return {
        ...state,
        count: state.count
          - state.step
      };
    case 'SET_STEP':
      return {
        ...state,
        step: action.payload
      };
    case 'RESET':
      return initialState;
    default:
      return state;
  }
};

  const [state, dispatch] =
    useReducer(reducer,
      initialState);

  return (
    <div>
      <p>Count: {state.count}</p>
      <p>Step: {state.step}</p>
      <button onClick={() =>
        dispatch({
          type: 'INCREMENT'
        })}>+</button>
      <button onClick={() =>
        dispatch({
          type: 'DECREMENT'
        })}>-</button>
      <input type="number"
        value={state.step}
        onChange={(e) =>
          dispatch({
            type: 'SET_STEP',
            payload:
              Number(e.target.value)
          })}
      />
      <button onClick={() =>
        dispatch({
          type: 'RESET'
        })}>Reset</button>

  );

  (key, initialValue) => {

        setResults(data);
      }, 500);

    // Cleanup: clear timer if
    // user types again within 500ms
    return () =>
      clearTimeout(timer);
  }, [query]);

  return (
    <div>
      <input value={query}
        onChange={e =>
          setQuery(e.target.value)}
        placeholder="Search users..."
      />
      {results.map(u => (
        <p key={u._id}>{u.name}</p>
      ))}
    </div>
  );
};
📋 Notes
// Without debounce:
// User types "amit"
// → 4 API calls: a, am, ami, amit

// With debounce (500ms):
// User types "amit"
// → 1 API call: amit
// → Waits until user stops typing

// How it works:
// 1. User types → query changes
// 2. useEffect runs, sets timer
// 3. User types again within 500ms
// 4. Cleanup clears old timer
// 5. New timer starts
// 6. After 500ms idle → API call

Q38. Theory Questions — Quick Answers
Q: What is React?
A: A JavaScript library for building user interfaces using components.

Q: What is the Virtual DOM?
A: An in-memory representation of the real DOM. React compares old vs new
   (diffing), then updates only what changed (reconciliation).

Q: What is JSX?
A: JavaScript XML — lets you write HTML-like syntax in JS. Compiles to
   React.createElement() calls via Babel.

Q: What are hooks?
A: Functions that let you use state and lifecycle features in functional
   components. useState, useEffect, useRef, useMemo, useCallback, etc.

Q: What is the difference between state and props?
A: State is internal, mutable (via setter). Props are external, read-only
   (passed from parent). Both trigger re-render when changed.

Q: What is useEffect?
A: Hook for side effects (API calls, timers, subscriptions). Runs after
   render. Dependency array controls when it re-runs.

Q: What is the Context API?
A: React's built-in way to share state globally without prop drilling.
   createContext → Provider (wraps app) → useContext (consume).

Q: What is React.memo?
A: Higher-order component that memoizes a component. Skips re-render if
   props haven't changed. Pairs with useCallback for function props.

Q: What is the key prop in lists?
A: Unique identifier that helps React efficiently update list items.
   Should be stable ID (not array index) for correct behavior.

Q: Controlled vs Uncontrolled components?
A: Controlled: value managed by React state (value + onChange).
   Uncontrolled: value managed by DOM (useRef).

Q: What is prop drilling? How to fix it?
A: Passing props through many intermediate components. Fix with Context API,
   Redux, or Zustand.

Q: What is React Router?
A: Library for client-side routing in SPAs. Maps URLs to components without
   page reload. Uses BrowserRouter, Routes, Route, Link.

Q: What is lifting state up?
A: Moving state to the nearest common parent when siblings need to share it.
   Parent holds state, passes down as props + callback functions.

Q: Class vs Functional components?
A: Functional (modern): simpler, uses hooks, recommended by React team.
   Class (legacy): uses this.state, lifecycle methods, more verbose.

Q: What are custom hooks?
A: Reusable functions starting with "use" that encapsulate hook logic.
   Example: useFetch, useAuth, useLocalStorage.

Q: useMemo vs useCallback?
A: useMemo caches a computed VALUE. useCallback caches a FUNCTION reference.
   Both take a dependency array and recompute only when deps change.

Q: What is lazy loading in React?
A: Loading components on demand using React.lazy() + Suspense.
   Splits bundle into chunks for faster initial page load.

Q: How does React handle forms?
A: Controlled components: value={state}, onChange={setState}.
   Single handler with [e.target.name] for multiple inputs.
Q39. Hooks Rules — Must Know!
// ======= RULES OF HOOKS =======

// 1. Only call hooks at the TOP LEVEL
// ❌ Inside if/else, for loops, nested functions
// ✅ Top of component function body

// 2. Only call hooks from REACT FUNCTIONS
// ✅ Functional components
// ✅ Custom hooks
// ❌ Regular JavaScript functions
// ❌ Class components

// 3. Hooks MUST be called in the SAME ORDER every render
// ❌ Conditionally calling hooks
if (loggedIn) {
  useEffect(() => {}) // ❌ WRONG!
}

// ✅ Put condition INSIDE the hook
useEffect(() => {
  if (loggedIn) { /* do something */ }
}, [loggedIn]);

// 4. Custom hooks must start with "use"
// ❌ getUsers()
// ✅ useUsers()

// INTERVIEW: "What are the rules of hooks?"
// → Only at top level, only in React functions,
//   same order every render, custom hooks start with "use".
