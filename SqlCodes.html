Section 10 — Window Functions (TCS Prime 🔥)

💡 ROW_NUMBER vs RANK: ROW_NUMBER gives unique sequential numbers. RANK gives same rank for ties and skips the next number.
Q35. ROW_NUMBER
Assigns a unique sequential number to each row ordered by salary descending.
💻 Query
SELECT name, salary,
  ROW_NUMBER() OVER (
    ORDER BY salary DESC
  ) AS rn
FROM employees;
📋 Output
name  |salary|rn
------|------|-
Suman |70000 |1
Amit  |60000 |2
Priya |55000 |3
Arjun |45000 |4
Ravi  |40000 |5
Neha  |35000 |6
(6 rows)
Q36. RANK
Same rank for ties, skips next. E.g., if two share rank 2, next is rank 4.
💻 Query
SELECT name, salary,
  RANK() OVER (
    ORDER BY salary DESC
  ) AS rnk
FROM employees;
📋 Output
name  |salary|rnk
------|------|-
Suman |70000 |1
Amit  |60000 |2
Priya |55000 |3
Arjun |45000 |4
Ravi  |40000 |5
Neha  |35000 |6
(6 rows)
-- No ties here, so same
-- as ROW_NUMBER.
Q36b. DENSE_RANK
Like RANK but does NOT skip numbers after ties. Very commonly compared in interviews.
💻 Query
SELECT name, salary,
  DENSE_RANK() OVER (
    ORDER BY salary DESC
  ) AS d_rnk
FROM employees;
📋 Output
name  |salary|d_rnk
------|------|-
Suman |70000 |1
Amit  |60000 |2
Priya |55000 |3
Arjun |45000 |4
Ravi  |40000 |5
Neha  |35000 |6
(6 rows)
-- No ties here. With ties:
-- RANK:  1,2,2,4
-- DENSE: 1,2,2,3

Q59. GRANT & REVOKE (DCL)
DCL — Data Control Language. Controls user permissions.
💻 Query
-- Grant SELECT permission
GRANT SELECT
ON employees TO 'user1';

-- Revoke permission
REVOKE SELECT
ON employees FROM 'user1';
📋 Output
Query OK

-- GRANT: user1 can now
-- run SELECT on employees.

-- REVOKE: permission
-- removed from user1.
Q60. TCL — COMMIT, ROLLBACK, SAVEPOINT
TCL — Transaction Control Language. Manages transactions.
💻 Query
START TRANSACTION;

UPDATE employees
SET salary = 80000
WHERE emp_id = 1;

SAVEPOINT sp1;

DELETE FROM employees
WHERE emp_id = 4;

-- Undo delete only:
ROLLBACK TO sp1;

-- Save the update:
COMMIT;
📋 Output
-- After COMMIT:
-- emp_id=1 salary → 80000 ✓
-- emp_id=4 NOT deleted ✓
--   (rolled back to sp1)

-- COMMIT = permanent save
-- ROLLBACK = undo changes
-- SAVEPOINT = partial undo

📝 Quick Reference — DDL vs DML vs DCL vs TCL:
DDL: CREATE, ALTER, DROP, TRUNCATE, RENAME
DML: SELECT, INSERT, UPDATE, DELETE
DCL: GRANT, REVOKE
TCL: COMMIT, ROLLBACK, SAVEPOINT
