Section 1 — Node.js Basics

Q1. What is Node.js?
Node.js is a JavaScript runtime built on Chrome's V8 engine. It allows running JS outside the browser (on server side).
💻 Key Points
// Node.js is:
// ✅ Single-threaded
// ✅ Non-blocking I/O
// ✅ Event-driven
// ✅ Asynchronous
// ✅ Uses V8 engine
// ✅ Package manager: npm

// Check version
node -v
npm -v
📋 Output
v20.11.0
10.2.0

// Node.js is NOT:
// ❌ A framework
// ❌ A programming language
// ❌ Multi-threaded (uses
//    event loop instead)

Section 3 — Node.js Core Modules

Q7. fs Module — File System
fs module handles file read/write. Always prefer async versions in production.
💻 Code
const fs = require('fs');

// ---- READ FILE ----
// Async (preferred)
fs.readFile('test.txt', 'utf8',
  (err, data) => {
    if (err) throw err;
    console.log(data);
});

// Sync (blocks event loop)
const data =
  fs.readFileSync('test.txt',
    'utf8');
console.log(data);

// ---- WRITE FILE ----
fs.writeFile('output.txt',
  'Hello World', (err) => {
    if (err) throw err;
    console.log('File written!');
});

// ---- APPEND ----
fs.appendFile('output.txt',
  '\nNew line', (err) => {
    if (err) throw err;
    console.log('Appended!');
});

// ---- DELETE ----
fs.unlink('output.txt',
  (err) => {
    if (err) throw err;
    console.log('Deleted!');
});
📋 Output
Hello from test.txt
File written!
Appended!
Deleted!

// Promise version:
const fs =
  require('fs').promises;

const data =
  await fs.readFile(
    'test.txt', 'utf8');

// Other methods:
// fs.mkdir()     → create dir
// fs.rmdir()     → remove dir
// fs.readdir()   → list files
// fs.rename()    → rename
// fs.existsSync()→ check exists
// fs.stat()      → file info

Q10. events Module — EventEmitter
EventEmitter is the core of Node's event-driven architecture. Many built-in modules inherit from it.
💻 Code
const EventEmitter =
  require('events');

const emitter = new EventEmitter();

// Register listener
emitter.on('userCreated',
  (user) => {
    console.log(
      'Welcome ' + user.name);
});

// Emit event
emitter.emit('userCreated',
  { name: 'Amit' });
📋 Output
Welcome Amit

// Key Methods:
// .on(event, callback)
//   → listen for event
// .emit(event, data)
//   → trigger event
// .once(event, callback)
//   → listen only once
// .removeListener()
//   → remove a listener
// .removeAllListeners()

Section 4 — HTTP Module (Without Express)

Q11. Create Basic HTTP Server
This is the raw Node.js way. Express simplifies all of this.
💻 Code
const http = require('http');

const server = http.createServer(
  (req, res) => {
    if (req.url === '/'
        && req.method === 'GET') {
      res.writeHead(200,
        {'Content-Type':
          'application/json'});
      res.end(
        JSON.stringify(
          { msg: "Hello!" }));
    }
    else if (req.url === '/about') {
      res.writeHead(200,
        {'Content-Type':
          'text/html'});
      res.end('<h1>About</h1>');
    }
    else {
      res.writeHead(404);
      res.end('Not Found');
    }
});

server.listen(3000, () => {
  console.log(
    'Server on port 3000');
});
📋 Output
Server on port 3000

// GET localhost:3000/
// → { "msg": "Hello!" }

// GET localhost:3000/about
// → <h1>About</h1>

// GET localhost:3000/xyz
// → Not Found (404)

// Problems without Express:
// ❌ Manual routing
// ❌ Manual body parsing
// ❌ No middleware support
// ❌ Verbose code
// → That's why we use Express!

Section 6 — Express Routing

Q14. HTTP Methods — GET, POST, PUT, DELETE
💻 Code
// GET — Read
app.get('/api/users',
  (req, res) => {
    res.json({ msg: "Get all" });
});

// POST — Create
app.post('/api/users',
  (req, res) => {
    res.status(201)
       .json({ msg: "Created" });
});

// PUT — Update (full)
app.put('/api/users/:id',
  (req, res) => {
    res.json({ msg: "Updated" });
});

// PATCH — Update (partial)
app.patch('/api/users/:id',
  (req, res) => {
    res.json({ msg: "Patched" });
});

// DELETE — Remove
app.delete('/api/users/:id',
  (req, res) => {
    res.json({ msg: "Deleted" });
});
📋 REST Convention
GET    /api/users
  → Get all users

GET    /api/users/:id
  → Get one user

POST   /api/users
  → Create new user

PUT    /api/users/:id
  → Replace entire user

PATCH  /api/users/:id
  → Update partial fields

DELETE /api/users/:id
  → Delete user

// This pattern is called
// RESTful API design.
// MOST asked in interviews!

Q18. Express Router — Modular Routes
Router splits routes into separate files. Essential for clean code.
💻 routes/userRoutes.js
const express = require('express');
const router = express.Router();

router.get('/', (req, res) => {
  res.json({ msg: "All users" });
});

router.get('/:id', (req, res) => {
  res.json({
    msg: "User " + req.params.id
  });
});

router.post('/', (req, res) => {
  res.status(201).json({
    msg: "Created",
    data: req.body
  });
});

router.put('/:id', (req, res) => {
  res.json({ msg: "Updated" });
});

router.delete('/:id',
  (req, res) => {
  res.json({ msg: "Deleted" });

Section 7 — Middleware 🔥 (Most Important Concept)

🔥 Middleware = functions that run BETWEEN request & response. They have access to req, res, and next(). This is the #1 most asked Express concept!
Q19. How Middleware Works
Request → Middleware 1 → Middleware 2 → Route Handler → Response
Each middleware MUST call next() to pass to the next one, or send a response.
💻 Code
// Custom middleware
const logger = (req, res, next) => {
  console.log(
    `${req.method} ${req.url}`
  );
  console.log(
    `Time: ${new Date()
      .toISOString()}`
  );
  next(); // MUST call next()
};

// Apply globally
app.use(logger);

app.get('/', (req, res) => {
  res.send('Home');
});
📋 Output
// When user visits /
GET /
Time: 2025-06-15T10:30:00Z

// If you forget next():
// Request hangs forever!
// Browser shows loading...

// Middleware types:
// 1. Application-level
//    (app.use)
// 2. Router-level
//    (router.use)
// 3. Built-in
//    (express.json, static)
// 4. Third-party
//    (cors, morgan, helmet)
// 5. Error-handling
//    (4 parameters)

Section 8 — Error Handling

Q23. Error Handling Middleware
Error middleware has 4 parameters: (err, req, res, next). Must be defined LAST.
💻 middleware/errorHandler.js
const errorHandler =
  (err, req, res, next) => {
    console.error(err.stack);

    const statusCode =
      res.statusCode === 200
        ? 500
        : res.statusCode;

    res.status(statusCode).json({
      message: err.message,
      stack:
        process.env.NODE_ENV
          === 'production'
          ? null
          : err.stack
    });
};

module.exports = errorHandler;
💻 Usage in server.js
const errorHandler =
  require('./middleware/
    errorHandler');

// All routes first...
app.use('/api/users', userRoutes);

// Error handler LAST
app.use(errorHandler);

// In routes, throw errors:
app.get('/api/test',
  (req, res) => {
    throw new Error(
      "Something broke!");
});

// Or with async:
app.get('/api/test',
  async (req, res, next) => {
    try {
      // risky code
    } catch (err) {
      next(err); // pass to handler
    }
});
Q24. Try-Catch & asyncHandler Wrapper
Instead of try-catch in every route, use a wrapper function.
💻 Code
// ---- asyncHandler utility ----
const asyncHandler = (fn) =>
  (req, res, next) =>
    Promise.resolve(fn(req, res, next))
      .catch(next);

// ---- Without wrapper ----
router.get('/', async (req, res,
  next) => {
    try {
      const users =
        await User.find();
      res.json(users);
    } catch (err) {
      next(err);
    }
});

// ---- With wrapper ---- ✅
router.get('/',
  asyncHandler(async (req, res) => {
    const users =
      await User.find();
    res.json(users);
}));
// No try-catch needed!
📋 Notes
// asyncHandler automatically
// catches any async errors
// and passes them to next()

// Or use npm package:
// npm i express-async-handler
//
// const asyncHandler =
//   require('express-async-handler');

// This is used in almost
// every production MERN app!

Section 11 — Full CRUD with MVC Pattern 🔥

Q28. Model — models/User.js
const mongoose = require('mongoose');

const userSchema = new mongoose.Schema({
  name: {
    type: String,
    required: [true, 'Name is required'],
    trim: true
  },
  email: {
    type: String,
    required: [true, 'Email is required'],
    unique: true,
    lowercase: true
  },
  password: {
    type: String,
    required: true,
    minlength: 6
  },
  role: {
    type: String,
    enum: ['user', 'admin'],
    default: 'user'
  }
}, {
  timestamps: true  // createdAt, updatedAt
});

module.exports = mongoose.model('User', userSchema);
Q29. Controller — controllers/userController.js
const User = require('../models/User');

// @desc   Get all users
// @route  GET /api/users
const getUsers = async (req, res) => {
  const users = await User.find().select('-password');
  res.json(users);
};

// @desc   Get single user
// @route  GET /api/users/:id
const getUser = async (req, res) => {
  const user = await User.findById(req.params.id).select('-password');
  if (!user) {
    return res.status(404).json({ msg: "User not found" });
  }
  res.json(user);
};

// @desc   Create user
// @route  POST /api/users
const createUser = async (req, res) => {
  const { name, email, password } = req.body;

Q32. Password Hashing — bcryptjs

// npm i bcryptjs
const bcrypt = require('bcryptjs');

// Hash password (Register)
const hashPassword =
  async (password) => {
    const salt =
      await bcrypt.genSalt(10);
    const hashed =
      await bcrypt.hash(
        password, salt);
    return hashed;
};

// Compare password (Login)
const comparePassword =
  async (entered, hashed) => {
    return await bcrypt.compare(
      entered, hashed);
};

// → $2a$10$X7kE...

  await comparePassword(
    '123456', hash);

// Salt rounds = 10 (standard)
// Higher = more secure but slower

//   → hashes password+salt
// bcrypt.compare(plain, hashed)
//   → returns true/false

Q34. Auth Middleware — Protect Routes
💻 middleware/auth.js
const jwt = require('jsonwebtoken');
const User = require('../models/User');

const protect =
  async (req, res, next) => {
    let token;

    if (req.headers.authorization
      && req.headers.authorization
        .startsWith('Bearer')) {

      try {
        // Get token from header
        token = req.headers
          .authorization.split(' ')[1];

        // Verify token
        const decoded = jwt.verify(
          token,
          process.env.JWT_SECRET
        );

        // Attach user to request
        req.user = await User
          .findById(decoded.id)
          .select('-password');

        next();
      } catch (error) {
        res.status(401).json({
          msg: 'Token invalid'
        });
      }
    }

    if (!token) {
      res.status(401).json({
        msg: 'No token provided'
      });
    }
};

module.exports = { protect };

Q35. Register & Login Controller
const User = require('../models/User');
const bcrypt = require('bcryptjs');
const jwt = require('jsonwebtoken');

const generateToken = (id) =>
  jwt.sign({ id }, process.env.JWT_SECRET, { expiresIn: '30d' });

// ======= REGISTER =======
const register = async (req, res) => {
  const { name, email, password } = req.body;

  // Check if user exists
  const userExists = await User.findOne({ email });
  if (userExists) {
    return res.status(400).json({ msg: 'User already exists' });
  }

  // Hash password
  const salt = await bcrypt.genSalt(10);
  const hashedPassword = await bcrypt.hash(password, salt);

  // Create user
  const user = await User.create({
    name, email, password: hashedPassword
  });

  res.status(201).json({
    _id: user._id,
    name: user.name,
    email: user.email,
    token: generateToken(user._id)
  });
};

// ======= LOGIN =======
const login = async (req, res) => {
  const { email, password } = req.body;

  // Find user
  const user = await User.findOne({ email });

  // Check password
  if (user && (await bcrypt.compare(password, user.password))) {
    res.json({
      _id: user._id,
      name: user.name,
      email: user.email,
      token: generateToken(user._id)
    });
  } else {
    res.status(401).json({ msg: 'Invalid credentials' });
  }
};

module.exports = { register, login };

Q36. Callbacks vs Promises vs Async/Await
This is a TOP interview question. Know all 3 patterns and when to use each.
💻 Code
// ===== 1. CALLBACK =====
const fs = require('fs');
fs.readFile('test.txt', 'utf8',
  (err, data) => {
    if (err) {
      console.log(err);
      return;
    }
    console.log(data);
});

// ===== 2. PROMISE =====
const readFile = (path) => {
  return new Promise(
    (resolve, reject) => {
      fs.readFile(path, 'utf8',
        (err, data) => {
          if (err) reject(err);
          else resolve(data);
      });
  });
};

readFile('test.txt')
  .then(data =>
    console.log(data))
  .catch(err =>
    console.log(err));

// ===== 3. ASYNC/AWAIT =====
const readData = async () => {
  try {
    const data =
      await readFile('test.txt');
    console.log(data);
  } catch (err) {
    console.log(err);
  }
};
readData();
📋 Comparison
// CALLBACK:
// ✅ Original Node pattern
// ❌ Callback hell (nested)
// ❌ Hard to read

// PROMISE:
// ✅ Chainable (.then)
// ✅ Better error handling
// ❌ Still chaining

// ASYNC/AWAIT:
// ✅ Cleanest syntax
// ✅ Looks synchronous
// ✅ Easy try/catch
// ✅ Used in Express routes

// INTERVIEW ANSWER:
// "I use async/await because
//  it's cleaner and easier
//  to debug. It's syntactic
//  sugar over Promises."

// Callback Hell:
// doA(() => {
//   doB(() => {
//     doC(() => {
//       doD(() => { ... })
//     })
//   })

Q37. Promise.all & Promise.race
💻 Code
// Promise.all — run in parallel
// Waits for ALL to finish
const [users, posts, comments] =
  await Promise.all([
    User.find(),
    Post.find(),
    Comment.find()
  ]);

// Promise.race — first to finish
const result =
  await Promise.race([
    fetch('/api/fast'),
    fetch('/api/slow')
  ]);
// Returns whichever resolves first

// Promise.allSettled
// Waits for all, even if some fail
const results =
  await Promise.allSettled([
    fetch('/api/a'),
    fetch('/api/b')
  ]);
// [{status:'fulfilled', value:...},
//  {status:'rejected', reason:...}]
📋 Notes
// Promise.all:
// → Runs all in parallel
// → Fails if ANY one fails
// → Great for multiple DB calls

// Promise.race:
// → Returns first settled
// → Useful for timeouts

// Promise.allSettled:
// → Never rejects
// → Returns all results
// → Good when some can fail

// INTERVIEW TIP:
// "I use Promise.all when
//  I need data from multiple
//  collections simultaneously
//  and all are required."
Q38. Event Loop — How Node.js Works
This is THE most asked Node.js theory question in interviews.
💻 Code Example

  console.log('2 - Timeout');

// Execution Order:
// 1. Call Stack (sync code)
//    → "Start", "End"
// 2. Microtask Queue
//    → nextTick (highest priority)
//    → Promises

//    → setTimeout, setInterval
//    → I/O callbacks

// Event Loop Phases:
// Timers → I/O → Idle →
// Poll → Check → Close

// KEY POINT:
// Microtasks run BEFORE
// macrotasks, even if
// setTimeout delay is 0!

Q39. Streams — Read Large Files
Streams process data chunk by chunk instead of loading everything into memory. Essential for large files.
💻 Code
const fs = require('fs');

// Read Stream
const readStream =
  fs.createReadStream(
    'bigfile.txt', 'utf8');

readStream.on('data',
  (chunk) => {
    console.log('Chunk received:',
      chunk.length, 'bytes');
});

readStream.on('end', () => {
  console.log('Done reading');
});

// Write Stream
const writeStream =
  fs.createWriteStream(
    'output.txt');

writeStream.write('Hello ');
writeStream.write('World');
writeStream.end();

// Pipe (read → write)
const src =
  fs.createReadStream('input.txt');
const dest =
  fs.createWriteStream('output.txt');
src.pipe(dest);
📋 Notes

Chunk received: 65536 bytes
Chunk received: 12340 bytes
Done reading

// 4 Types of Streams:
// 1. Readable  (fs.createReadStream)
// 2. Writable  (fs.createWriteStream)
// 3. Duplex    (read + write, TCP)
// 4. Transform (modify data, gzip)

// .pipe() connects streams
// src.pipe(dest)
// → reads from src, writes to dest

// Why Streams?
// ❌ fs.readFile → loads ALL into RAM
// ✅ Stream → processes in chunks
// → Better for large files,
//   video streaming, etc.

Section 19 — Top Interview Questions 🔥

Q45. Theory Questions — Quick Answers
Q: What is Node.js?
A: A JavaScript runtime built on V8 engine for server-side programming.

Q: Why is Node.js single-threaded?
A: It uses event loop & non-blocking I/O for concurrency without threads.

Q: What is the Event Loop?
A: Mechanism that handles async callbacks. Processes microtasks (Promises)
   before macrotasks (setTimeout).

Q: What is middleware in Express?
A: Functions with (req, res, next) that execute between request and response.
   Must call next() or send response.

Q: Difference between app.use() and app.get()?
A: app.use() matches ALL HTTP methods, app.get() only matches GET.

Q: What is CORS?
A: Cross-Origin Resource Sharing. Browser blocks requests from different
   origins by default. cors() middleware enables it.

Q: What is JWT?
A: JSON Web Token — a signed token containing user info, used for
   stateless authentication. Has header, payload, signature.

Q: Difference between SQL and NoSQL?
A: SQL: structured tables, relations, schema. NoSQL: flexible documents,
   no schema required, horizontal scaling.

Q: What is req.params vs req.query vs req.body?
A: params = /users/:id (URL path), query = ?key=val (URL params),
   body = POST/PUT data (JSON payload).

Q: How do you handle errors in Express?
A: Use try-catch / asyncHandler wrapper + error middleware with
   4 params (err, req, res, next) defined last.

Q: What is the difference between PUT and PATCH?
A: PUT replaces entire resource. PATCH updates partial fields.

Q: What is process.env?
A: Object containing environment variables. Used to store secrets
   like DB URI, JWT secret. Loaded from .env file using dotenv.

Q: What is Mongoose?
A: ODM (Object Data Modeling) library for MongoDB + Node.js.
   Provides schema, validation, middleware, and query building.

Q: CommonJS vs ES Modules?
A: CommonJS: require/module.exports (default in Node).
   ES Modules: import/export (needs "type":"module" in package.json).

Q: What is npm?
A: Node Package Manager. Installs, manages, publishes packages.
   package.json tracks dependencies. node_modules stores them.

📝 Express Quick Cheat:
app.use() → global middleware  |  app.get/post/put/delete() → route handlers
req.params → :id in URL  |  req.query → ?key=val  |  req.body → POST data
res.json() → send JSON  |  res.status() → set code  |  next() → pass to next middleware
express.json() → parse body  |  express.static() → serve files  |  express.Router() → modular routes
