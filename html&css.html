5. Links & Images
Links use <a>, images use <img>. Always add alt for accessibility.
<!-- Links -->
<a href="https://google.com">Google</a>
<a href="https://google.com" target="_blank">Open in New Tab</a>
<a href="#section1">Jump to Section 1</a>
<a href="mailto:test@mail.com">Email Us</a>
<a href="tel:+1234567890">Call Us</a>
<a href="file.pdf" download>Download PDF</a>

<!-- Images -->
<img src="photo.jpg" alt="Description" width="300" height="200">

<!-- Image as Link -->
<a href="/page">
    <img src="logo.png" alt="Logo">
</a>

<!-- Responsive Image -->
<picture>
    <source media="(max-width:600px)" srcset="small.jpg">

⚠️ target="_blank" should always use rel="noopener noreferrer" for security.

Meta tags go inside <head>. Critical for SEO and social sharing.

13. Accessibility (a11y) Best Practices
Make websites usable for everyone, including people with disabilities.
<!-- 1. Always use alt text -->
<img src="cat.jpg" alt="Orange cat sleeping on a couch">

<!-- 2. Label form inputs -->
<label for="email">Email:</label>
<input type="email" id="email" name="email">

<!-- 3. Use ARIA roles when needed -->
<div role="navigation" aria-label="Main menu">...</div>
<button aria-expanded="false" aria-controls="menu">Menu</button>

<!-- 4. Skip navigation link -->
<a href="#main-content" class="skip-link">Skip to content</a>

<!-- 5. Use semantic HTML (biggest a11y win!) -->
<nav> instead of <div class="nav">

<!-- 6. Tab order -->
<button tabindex="0">Focusable</button>
<div tabindex="-1">Programmatically focusable only</div>

<!-- 7. Language attribute -->
<html lang="en">
ARIA AttributePurpose
roleDefine element role (button, navigation, alert)
aria-labelInvisible label for screen readers
aria-hidden="true"Hide from screen readers
aria-expandedExpanded/collapsed state
aria-requiredMark as required
aria-live="polite"Announce dynamic content changes

3. Specificity & Cascade
When multiple rules target the same element, specificity determines which wins.
/* Specificity Calculation: (a, b, c, d)
   a = inline styles
   b = ID selectors
   c = class / pseudo-class / attribute selectors
   d = element / pseudo-element selectors
*/

*                    →  (0,0,0,0)
p                    →  (0,0,0,1)
p.class              →  (0,0,1,1)
p#id                 →  (0,1,0,1)
#id .class p         →  (0,1,1,1)
style="..."          →  (1,0,0,0)
!important           →  Overrides everything! ⚠️
💻 Example
/* Who wins? */
p { color: blue; }           /* 0,0,0,1 */
.text { color: green; }      /* 0,0,1,0 ✅ */
#intro { color: red; }       /* 0,1,0,0 ✅✅ */

<p id="intro" class="text">
    This text is RED
</p>
📊 Specificity Hierarchy
!important     → 🔴 HIGHEST (avoid!)
inline style   → (1,0,0,0)
#id            → (0,1,0,0)
.class         → (0,0,1,0)
element        → (0,0,0,1)
*              → (0,0,0,0)
inherited      → 🟢 LOWEST
⚠️ Avoid !important — it makes debugging very hard. Use specificity instead.
4. Box Model — The Foundation
Every HTML element is a box with: content → padding → border → margin.
💻 Code
.box {
    width: 200px;
    height: 100px;
    padding: 20px;
    border: 5px solid #333;
    margin: 10px;

    /* Without this: total = 200+40+10 = 250px wide! */
    /* With this: total = 200px (padding+border inside) */
    box-sizing: border-box;
}

/* Best practice: apply to ALL elements */
*, *::before, *::after {
    box-sizing: border-box;
}
📐 Box Model Diagram
┌─── Margin (10px) ──────────────┐
│ ┌─── Border (5px) ───────────┐ │
│ │ ┌─── Padding (20px) ─────┐ │ │
│ │ │                         │ │ │
│ │ │      CONTENT            │ │ │
│ │ │    200px × 100px        │ │ │
│ │ │                         │ │ │
│ │ └─────────────────────────┘ │ │
│ └─────────────────────────────┘ │
└─────────────────────────────────┘

content-box (default):
  Total width = width + padding + border
  = 200 + 40 + 10 = 250px

border-box (recommended):
  Total width = width = 200px
  (padding+border fit inside)
💡 Always use box-sizing: border-box globally!

7. Flexbox — Complete Guide
Flexbox is a one-dimensional layout system (row OR column). Parent = flex container, Children = flex items.
💻 Container Properties
.container {
    display: flex;

    /* Direction */
    flex-direction: row;          /* default → */
    flex-direction: row-reverse;  /* ← */
    flex-direction: column;       /* ↓ */
    flex-direction: column-reverse; /* ↑ */

    /* Wrap */
    flex-wrap: nowrap;     /* default: squeeze */
    flex-wrap: wrap;       /* wrap to next line */
    flex-wrap: wrap-reverse;

    /* Shorthand */
    flex-flow: row wrap;

    /* Main axis alignment */
    justify-content: flex-start;   /* |||_____ */
    justify-content: flex-end;     /* _____|||  */
    justify-content: center;       /* ___|||___ */
    justify-content: space-between;/* |__|__|__ */
    justify-content: space-around; /* _|__|__|_ */
    justify-content: space-evenly; /* _|_|_|_|_ */

    /* Cross axis alignment */
    align-items: stretch;     /* default: fill */
    align-items: flex-start;  /* top */
    align-items: flex-end;    /* bottom */
    align-items: center;      /* center */
    align-items: baseline;    /* text baseline */

    /* Multi-line cross alignment */
    align-content: flex-start;
    align-content: center;
    align-content: space-between;

    /* Gap between items */
    gap: 10px;
    row-gap: 10px;
    column-gap: 20px;
}
💻 Item Properties
.item {
    /* Growth ratio */
    flex-grow: 0;    /* default: don't grow */
    flex-grow: 1;    /* take available space */

    /* Shrink ratio */
    flex-shrink: 1;  /* default: can shrink */
    flex-shrink: 0;  /* don't shrink */

    /* Base size before grow/shrink */
    flex-basis: auto;   /* default */
    flex-basis: 200px;
    flex-basis: 25%;

    /* Shorthand: grow shrink basis */
    flex: 0 1 auto;     /* default */
    flex: 1;             /* = 1 1 0% (grow!) */
    flex: 1 0 200px;

    /* Override align-items for this item */
    align-self: center;
    align-self: flex-end;

    /* Order (default: 0) */
    order: -1;   /* move first */
    order: 1;    /* move last */
}
💡 Common patterns:
/* Center everything (holy grail!) */
.center-all {
    display: flex;
    justify-content: center;
    align-items: center;
    height: 100vh;
}

/* Navbar: logo left, links right */
.navbar {
    display: flex;
    justify-content: space-between;
    align-items: center;
    padding: 10px 20px;
}

/* Equal width columns */
.columns {
    display: flex;
    gap: 20px;
}
.columns > * {
    flex: 1;    /* each takes equal space */
}

/* Sticky footer */
body {
    display: flex;
    flex-direction: column;

Grid is a two-dimensional layout system (rows AND columns simultaneously).

🟠 HTML Interview Questions

HTML Questions (1–25)
HTMLEasy Q1. What is HTML?
HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure of web content using elements/tags. It is not a programming language — it doesn't have logic, loops, or conditions.
HTMLEasy Q2. What is the difference between HTML and HTML5?
HTML5 is the latest version with new features:
• Semantic tags: header, nav, main, section, article, aside, footer
• New input types: email, date, color, range, url, search
• Media: <audio>, <video> without plugins
• Canvas & SVG for graphics
• APIs: Geolocation, Web Storage, Web Workers, Drag & Drop
• Simpler doctype: <!DOCTYPE html>
HTMLEasy Q3. What are semantic elements? Why use them?
Semantic elements clearly describe their meaning to both the browser and the developer. Examples: <header>, <nav>, <main>, <article>, <section>, <footer>.

Benefits: Better SEO, improved accessibility (screen readers), cleaner and more readable code, easier maintenance.
HTMLEasy Q4. What is the difference between <div> and <span>?
<div> is a block-level element — takes full width, starts on a new line. Used for grouping large sections.
<span> is an inline element — takes only the needed width, stays in the same line. Used for styling small text portions.
HTMLEasy Q5. What is the difference between <b> and <strong>?
<b> — Visually bold only. No semantic meaning.
<strong> — Bold with semantic importance. Screen readers may emphasize it. Preferred for SEO and accessibility.
HTMLEasy Q6. What is the difference between <i> and <em>?
<i> — Visually italic, no semantic meaning (used for icons, technical terms).
<em> — Italic with emphasis. Screen readers change tone. Semantic meaning.
HTMLMedium Q7. What is the difference between id and class?
id: Unique — only ONE element per page. Higher specificity. Used for unique elements. Selected with #id.
class: Reusable — multiple elements can share. Lower specificity. Used for styling groups. Selected with .class.
An element can have multiple classes but only one id.
HTMLEasy Q8. What are void/self-closing elements?
Elements that don't have a closing tag and can't have children:
<br>, <hr>, <img>, <input>, <meta>, <link>, <area>, <col>, <source>, <embed>, <wbr>
HTMLMedium Q9. What is the difference between GET and POST?
GET: Data in URL, visible, limited length (~2048 chars), bookmarkable, cached, for fetching data.
POST: Data in body, hidden from URL, no size limit, not cached, for sending/modifying data (forms, passwords).
HTMLMedium Q10. What is the difference between localStorage and sessionStorage?
Both store key-value pairs in the browser (5-10MB).
localStorage: Persists forever (until manually cleared). Shared across tabs of same origin.
sessionStorage: Cleared when tab closes. Per-tab storage, not shared.
HTMLEasy Q11. What is the purpose of the alt attribute in images?
• Provides alternative text when image fails to load
• Read by screen readers for accessibility
• Helps with SEO — search engines understand image content
• Required by WCAG accessibility standards
HTMLMedium Q12. What is the difference between <link> and <a>?
<link>: Used in <head> to link external resources (CSS, favicon, fonts). Not visible. Self-closing.
<a>: Anchor tag for hyperlinks. Visible and clickable. Used in <body>.
HTMLEasy Q13. What is the viewport meta tag?
<meta name="viewport" content="width=device-width, initial-scale=1.0">

It tells the browser to match the screen's width and set initial zoom to 100%. Essential for responsive design — without it, mobile browsers render at desktop width (~980px) and zoom out.
HTMLMedium Q14. What are data-* attributes?
Custom attributes to store extra data on HTML elements without non-standard attributes.
<div data-user-id="123" data-role="admin">
Access via JS: element.dataset.userId, element.dataset.role
Valid, won't conflict with future HTML attributes.
HTMLMedium Q15. What is the difference between <script>, <script async>, and <script defer>?
<script>: Blocks HTML parsing. Downloads & executes immediately.
<script async>: Downloads in parallel, executes as soon as ready (may interrupt parsing). Order not guaranteed.
<script defer>: Downloads in parallel, executes after HTML parsing. Order guaranteed. Best for most scripts!
HTMLMedium Q16. What is the difference between <section> and <div>?
<section>: Semantic — represents a thematic grouping of content, typically with a heading. Meaningful to screen readers and search engines.
<div>: Non-semantic — generic container with no meaning. Use when no semantic element fits.
HTMLEasy Q17. What is the difference between <article> and <section>?
<article>: Self-contained, independently distributable content (blog post, news article, comment, product card).
<section>: Thematic grouping that's part of a larger whole (chapter, tab panel, grouped related content).
Rule: If content makes sense on its own (in an RSS feed) → article. Otherwise → section.
HTMLHard Q18. What is Shadow DOM?
Shadow DOM is a web standard that provides encapsulation for DOM and CSS. It allows attaching a "hidden" DOM tree to an element. Styles inside Shadow DOM don't leak out, and external styles don't leak in. Used in Web Components. Example: browser's default <video> controls use Shadow DOM internally.
HTMLMedium Q19. What is the difference between disabled and readonly?
disabled: Can't interact, grayed out, value NOT submitted with form.
readonly: Can't edit but can select/copy, value IS submitted with form. Only works on text inputs and textareas.
HTMLMedium Q20. What are Web Workers?
Web Workers run JavaScript in a background thread, separate from the main thread. This prevents heavy computations from blocking the UI. They communicate with the main thread via postMessage(). They cannot access the DOM directly.
HTMLEasy Q21. What is the <!DOCTYPE html> declaration?
It tells the browser which version of HTML the page uses. <!DOCTYPE html> activates standards mode (HTML5). Without it, browsers use quirks mode which emulates old, buggy behavior for backward compatibility. Always include it at the very top.
HTMLMedium Q22. What is the difference between Canvas and SVG?
Canvas: Pixel-based (raster), drawn via JavaScript, better for games/complex animations, doesn't scale well, single DOM element.
SVG: Vector-based, defined in XML/HTML, scales perfectly (any resolution), each shape is a DOM element (can attach events), better for icons/charts/logos.
HTMLMedium Q23. What is CORS?
Cross-Origin Resource Sharing — a security mechanism that allows/restricts web pages from making requests to a different domain. Browsers block cross-origin requests by default. The server must send Access-Control-Allow-Origin headers to allow it.
HTMLEasy Q24. What is the difference between colspan and rowspan?
colspan: Makes a table cell span across multiple columns horizontally.
rowspan: Makes a table cell span across multiple rows vertically.
HTMLMedium Q25. What are ARIA roles and when should you use them?
ARIA (Accessible Rich Internet Applications) roles define what an element is or does for assistive technologies. Examples: role="button", role="navigation", role="alert".

Rule: Use semantic HTML first (it has built-in ARIA). Only add ARIA roles when you're building custom interactive widgets that can't be represented with native HTML elements.

CSSEasy Q26. What is the CSS Box Model?
Every HTML element is a rectangular box with 4 layers (inside out):
Content → Padding → Border → Margin

With box-sizing: content-box (default), width/height apply to content only; padding and border are added extra.
With box-sizing: border-box, width/height include padding and border.
CSSMedium Q27. What is CSS specificity? How is it calculated?
Specificity determines which CSS rule wins when multiple rules target the same element. Calculated as (a, b, c, d):
a = inline styles (1,0,0,0)
b = ID selectors (0,1,0,0)
c = class / pseudo-class / attribute selectors (0,0,1,0)
d = element / pseudo-element selectors (0,0,0,1)
Higher value wins. !important overrides everything (avoid!).
CSSEasy Q28. What is the difference between margin and padding?
Padding: Space inside the border, between content and border. Background color applies to padding. Always positive.
Margin: Space outside the border, between the element and its neighbors. Transparent. Can be negative. Vertical margins collapse (overlap).
CSSMedium Q29. What is margin collapse?
When two vertical margins touch, they merge into one (the larger value wins, they don't add up).
Example: Element A has margin-bottom: 20px, Element B has margin-top: 30px. The gap between them is 30px, not 50px.

Doesn't happen: with horizontal margins, flex/grid items, floated elements, or elements with padding/border between them.
CSSEasy Q30. What is the difference between display: none and visibility: hidden?
display: none — Element is completely removed from layout. Takes no space. Like it doesn't exist.
visibility: hidden — Element is invisible but still occupies space in the layout.
opacity: 0 — Invisible, occupies space, and is still clickable/interactive.
CSSMedium Q31. Explain the position property values.
static: Default. Normal flow. top/left/z-index don't work.
relative: Offset from its normal position. Space is preserved.
absolute: Positioned relative to nearest positioned ancestor. Removed from flow.
fixed: Positioned relative to viewport. Stays on scroll. Removed from flow.
sticky: Acts like relative until scroll threshold, then acts like fixed.
CSSMedium Q32. What is Flexbox? Explain main and cross axis.
Flexbox is a one-dimensional layout system for arranging items in a row or column.

Main axis: Direction items flow (default: horizontal →). Controlled by justify-content.
Cross axis: Perpendicular to main axis (default: vertical ↓). Controlled by align-items.

flex-direction: column swaps the axes.
CSSMedium Q33. What is CSS Grid? How is it different from Flexbox?
Grid is a two-dimensional layout system (rows AND columns simultaneously).

Flexbox: 1D, content-driven, best for components (navbars, card rows).
Grid: 2D, layout-driven, best for page structure (full layouts, dashboards).

They complement each other — use Grid for the page layout, Flexbox for components inside.
CSSEasy Q34. What are pseudo-classes and pseudo-elements?
Pseudo-classes (:) — Style based on state: :hover, :focus, :active, :first-child, :nth-child(), :not()
Pseudo-elements (::) — Style a specific part: ::before, ::after, ::first-line, ::first-letter, ::selection, ::placeholder

::before and ::after create virtual elements (require content property).
CSSEasy Q35. What is a media query?
Media queries apply CSS rules based on device characteristics (screen width, orientation, color scheme, etc.).
@media (max-width: 768px) { ... } — Applies when screen is ≤768px.
Mobile-first: Use min-width (start small, add complexity).
Desktop-first: Use max-width (start large, simplify).
CSSEasy Q36. What are the different CSS units?
Absolute: px (pixels)
Relative:
• em — relative to parent's font-size
• rem — relative to root (html) font-size
• % — relative to parent
• vw/vh — 1% of viewport width/height
• fr — fraction (Grid only)
• ch — width of "0" character

Best practice: Use rem for font sizes, px for borders, % or vw for widths.
CSSMedium Q37. What is the difference between em and rem?
em: Relative to the parent element's font-size. Compounds (nesting multiplies). Can be unpredictable.
rem: Relative to the root (html) font-size. Consistent and predictable regardless of nesting.

If html is 16px: 1.5rem = 24px everywhere. But 1.5em inside a 20px parent = 30px.
CSSMedium Q38. How do you center a div horizontally and vertically?
Method 1 — Flexbox (recommended):
display: flex; justify-content: center; align-items: center; height: 100vh;

Method 2 — Grid:
display: grid; place-items: center; height: 100vh;

Method 3 — Position + Transform:
position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%);

Method 4 — Margin auto (horizontal only):
width: 300px; margin: 0 auto;
CSSMedium Q39. What is the z-index and how does it work?
z-index controls the stacking order of overlapping elements. Higher value = on top.
Only works on positioned elements (relative, absolute, fixed, sticky).
Creates a stacking context — z-index values are relative within that context. A child can never appear above a parent's sibling with a higher z-index.
CSSMedium Q40. What is the difference betweentransition and animation?
Transition: Animates between two states (e.g., hover). Needs a trigger (hover, focus, class change). Simple: property, duration, timing.

Animation: Multi-step with @keyframes. Can run automatically, loop infinitely, have multiple stages (0%, 50%, 100%). More complex and powerful.

Rule of thumb: Use transitions for simple hover/state changes. Use animations for complex, multi-step, or auto-playing effects.
CSSMedium Q41. What is the difference between transform, transition, and animation?
transform: Changes an element's shape/position (rotate, scale, translate, skew). Instant, no motion by itself.
transition: Adds smooth motion between two states when a property changes.
animation: Creates complex multi-step motion using @keyframes. Can auto-play and loop.

Transform = WHAT changes. Transition/Animation = HOW it changes over time.
CSSMedium Q42. What is a CSS preprocessor? Name some.
A CSS preprocessor extends CSS with features like variables, nesting, mixins, functions, loops, then compiles to regular CSS.

Popular preprocessors:
• SASS/SCSS — Most popular. Variables ($color), nesting, mixins, partials.
• LESS — Similar to SASS, uses @variables.
• Stylus — Flexible syntax, optional semicolons/braces.

Note: Modern CSS now has native variables (--var) and nesting support.
CSSMedium Q43. What is BEM methodology?
BEM = Block Element Modifier — a CSS naming convention for maintainable code.

Block: Standalone component → .card
Element: Part of a block → .card__title, .card__image
Modifier: Variation → .card--featured, .card__title--large

Benefits: Avoids naming conflicts, self-documenting, flat specificity (all classes).
CSSEasy Q44. What is the difference between max-width and min-width?
max-width: Element can be at most this wide. Shrinks on smaller screens. Used for desktop-first responsive design.
min-width: Element is at least this wide. Grows on larger screens. Used for mobile-first responsive design.

Example: .container { width: 90%; max-width: 1200px; } — Fluid but caps at 1200px.
CSSHard Q45. What is a stacking context?
A stacking context is a 3D conceptual layer where z-index values are relative to that context only.

Created by: positioned element with z-index, opacity < 1, transform, filter, will-change, isolation: isolate.

Key insight: A child's z-index is confined to its parent's stacking context. Even z-index: 999999 won't escape a parent with z-index: 1.
CSSMedium Q46. What is the difference between inline, inline-block, and block?
inline: No width/height, no top/bottom margin. Flows with text. (span, a, strong)
inline-block: Like inline (same line) but accepts width/height and all margins.
block: Full width, new line, accepts all properties. (div, p, h1)
CSSEasy Q47. How do you make a website responsive?
1. Add viewport meta tag
2. Use relative units (%, rem, vw) instead of fixed px
3. Use Flexbox/Grid for layouts
4. Add media queries for breakpoints
5. Use max-width: 100% on images
6. Use clamp() for fluid typography
7. Mobile-first approach (min-width queries)
8. Test on multiple devices/screen sizes
CSSMedium Q48. What are CSS variables? How do they differ from SASS variables?
CSS Variables (--name): Native, runtime, can be changed with JS, cascade/inherit, scoped to selectors, work in media queries.
SASS Variables ($name): Compile-time only, can't change at runtime, don't cascade, global or local scope.

CSS variables are more powerful because they're live — you can update them with JavaScript or media queries for dark mode, theming, etc.
CSSMedium Q49. What is the will-change property?
will-change hints to the browser that an element will be animated, so it can optimize rendering (e.g., promote to GPU layer).
will-change: transform, opacity;

Warning: Overuse wastes memory. Only apply to elements that will actually animate. Remove when animation is done.
CSSHard Q50. What is the difference between reflow and repaint?
Reflow (Layout): Browser recalculates position and size of elements. Expensive. Triggered by: changing width, height, margin, padding, font-size, adding/removing elements.

Repaint: Browser redraws visual appearance without layout change. Cheaper. Triggered by: changing color, background, visibility, box-shadow.

Best practice: Animate only transform and opacity — they don't trigger reflow or repaint (GPU-accelerated).
CSSEasy Q51. What is the difference between absolute and relative units?
Absolute units: Fixed size regardless of context. px, cm, mm, in, pt, pc. Pixel (px) is most common.
Relative units: Size depends on something else. em (parent font), rem (root font), % (parent size), vw/vh (viewport), fr (grid fraction).

Relative units are essential for responsive design.
CSSMedium Q52. How does CSS float work? Why is it considered legacy?
float moves an element to the left or right, allowing text to wrap around it. Originally designed for text wrapping around images.

It was misused for page layouts before Flexbox/Grid existed. Problems: parent collapse (needs clearfix), hard to center, unpredictable behavior.

Modern alternative: Use Flexbox or Grid for layouts. Float only for its original purpose (text wrapping).
CSSMedium Q53. What is clamp() in CSS?
clamp(min, preferred, max) creates a responsive value that scales between a minimum and maximum.

font-size: clamp(1rem, 2.5vw, 3rem);
— At least 1rem, ideally 2.5vw, at most 3rem.

Eliminates the need for many media queries. Works for font-size, width, padding, etc.
CSSHard Q54. What is the contain property?
contain tells the browser that an element is independent from the rest of the page, allowing rendering optimizations.

contain: layout; — Layout changes inside don't affect outside.
contain: paint; — Content doesn't render outside bounds.
contain: size; — Element's size doesn't depend on children.
contain: content; — Shorthand for layout + paint.
contain: strict; — All containment types.

Useful for performance in complex pages with many components.
CSSMedium Q55. What is the difference between @import and <link> for CSS?
<link>: Loads CSS in parallel. Recommended. Better performance.
@import: Loads CSS sequentially (blocks rendering). Slower. Can cause FOUC (Flash of Unstyled Content).

Always prefer <link> in the HTML <head>. @import is acceptable inside CSS files for organization but avoid in <style> tags.

🔥 Scenario-Based & Tricky Questions

Scenario & Tricky Questions (56–65)
HTMLCSSHard Q56. How would you improve the performance of a web page?
HTML: Minimize DOM depth, use semantic tags, lazy-load images (loading="lazy"), use defer/async for scripts.
CSS: Minimize CSS, remove unused styles, avoid @import, use will-change sparingly, animate only transform/opacity, use contain property.
Images: Use WebP/AVIF formats, responsive images (srcset), compress images, use CDN.
General: Enable gzip/brotli compression, use caching headers, minimize HTTP requests, use critical CSS inline.
CSSHard Q57. Why would you use transform: translateZ(0)?
It's a hack to promote an element to its own GPU layer (hardware acceleration). This can make animations smoother because the browser composites the layer on the GPU instead of repainting on the CPU.

Modern alternative: will-change: transform;
Warning: Too many GPU layers waste memory. Use only when needed.
CSSMedium Q58. How do you create a triangle using only CSS?
Use borders on an element with zero width/height:
.triangle {
    width: 0;
    height: 0;
    border-left: 50px solid transparent;
    border-right: 50px solid transparent;
    border-bottom: 100px solid red;
}
The transparent borders create the angled sides, and the colored border forms the visible triangle face.
HTMLCSSMedium Q59. How do you make a sticky footer that stays at the bottom?
Flexbox method: 
body {
    display: flex;
    flex-direction: column;
    min-height: 100vh;
}
main { flex: 1; }
/* footer stays at bottom naturally */
Grid method: 
body {
    display: grid;
    grid-template-rows: auto 1fr auto;
    min-height: 100vh;
}
CSSMedium Q60. How do you create a responsive square?
Modern (aspect-ratio):
.square { width: 50%; aspect-ratio: 1 / 1; }

Legacy (padding trick):
.square { width: 50%; padding-bottom: 50%; height: 0; }
(Padding % is relative to the element's width, so equal width and padding-bottom = square.)
HTMLMedium Q61. What happens if you don't include <!DOCTYPE html>?
The browser enters quirks mode, emulating bugs and behaviors of old browsers (IE5). This causes inconsistent rendering: box model behaves differently, margins/padding calculate oddly, and modern CSS features may not work correctly. Always include it.
CSSHard Q62. How would you implement dark mode?
Step 1: Define colors as CSS variables:
:root {
    --bg: #fff; --text: #333;
}
[data-theme="dark"] {
    --bg: #1a1a2e; --text: #eee;
}
body { background: var(--bg); color: var(--text); }
Step 2: Toggle with JS: document.documentElement.dataset.theme = 'dark';
Step 3: Auto-detect OS preference: @media (prefers-color-scheme: dark) { ... }
Step 4: Save preference in localStorage.
CSSMedium Q63. What is the difference between :nth-child() and :nth-of-type()?
:nth-child(n): Selects the nth child regardless of type. p:nth-child(2) selects the 2nd child IF it's a <p>. If the 2nd child is a <div>, nothing is selected.

:nth-of-type(n): Selects the nth element of that specific type. p:nth-of-type(2) selects the 2nd <p> regardless of other siblings between them.
HTMLMedium Q64. Can you have multiple <header> or <footer> elements?
Yes! You can have multiple <header> and <footer> elements — one for each sectioning element (article, section, aside). They don't have to be unique to the page.

However, there should only be one <main> element per page.
CSSHard Q65. What is FOUC and how do you prevent it?
FOUC = Flash of Unstyled Content — the page briefly appears without CSS before styles load.

Causes: @import in CSS, JS-loaded stylesheets, slow CSS delivery, render-blocking scripts before CSS.

Prevention:
• Use <link> in <head> (not @import)
• Inline critical CSS
• Use preload: <link rel="preload" href="style.css" as="style">
• Put scripts at bottom or use defer
